# IEEE Paper Extractor

这是一个针对 IEEE Xplore HTML 论文页的 Chrome / Edge 扩展。它的目标不是只复制公式，而是把当前论文页面尽量整理成可读的文字版导出来。

## 能做什么

- 提取标题、作者、摘要。
- 顺序提取正文段落、小节标题、列表。
- 尽量把公式转成 LaTeX；拿不到时回退为 MathML 或可见文本。
- 提取图表题注。
- 提取参考文献。
- 复制为 Markdown 或纯文本。
- 直接下载为 `.md` 或 `.txt`。

## 适用范围

优先面向 **IEEE Xplore 的 HTML 正文页**。如果你打开的是摘要页、PDF 内嵌页，或者页面正文尚未加载完全，提取效果会差很多。

## 安装

1. 打开 Chrome 或 Edge。
2. 进入扩展页：
   - Chrome: `chrome://extensions`
   - Edge: `edge://extensions`
3. 打开“开发者模式”。
4. 选择“加载已解压的扩展程序”。
5. 选中本目录。

## 使用

打开 IEEE Xplore 论文 HTML 页面后，点击浏览器工具栏中的扩展图标：

- `复制全文 Markdown`：适合后续整理、喂给笔记软件。
- `复制全文纯文本`：适合直接阅读或粘贴。
- `下载 Markdown / 下载纯文本`：保存到本地。
- `复制本页全部公式`：单独导出公式。
- `打开页内面板`：在当前页面右上角展示提取结果。

## 当前实现思路

扩展会先猜测论文正文的主内容容器，再按文档顺序遍历标题、段落、列表、表格、图注和公式节点，最后把结果拼成 Markdown / TXT。对于公式，会优先尝试页面中已有的 TeX / MathML / MathJax 源，而不是 OCR。

## 局限

- 不是“恢复出版社原始排版”的工具，而是“提取可读文字内容”的工具。
- 不保证 100% 无重复或无噪声；不同年份、不同期刊、不同前端版本的 IEEE 页面结构差异较大。
- 某些现代动态页面如果把正文分块懒加载，可能需要先滚动到页面后部，再点提取。
- PDF 视图不在核心支持范围内。
